WebMail PHP administration

WebMail PHP administration performs just through the single Administration Console.

Navigate to "http://path_to_webmail/mailadm.php" page in your browser. Use "mailadm" as Login name.
Type your default admin password "12345". It is strongly recommended to change this password.

After login Administration Console appears with the following menu:

 

Administration Console:

Value Description
WebMail Settings Use this section to set settings of WebMail system, such as Site Name, POP3/SMTP Servers and other default settings.
Interface Settings Customize WebMail's look and feel: skin, etc.
Login Settings Customize login window look and functionality.
Debug Settings Customize logging and other debug settings.
Help Opens WebMail documentation.
Exit Exits WebMail administration panel, and returns to the administration login page.

 

WebMail Settings section.

Here you can set some parameters and default values.

Value Description
Site Name Title that will be shown in browser's header.
Incoming Mail Incoming mail server name or IP, such as "pop.server.com".
Incoming Mail Port Incoming mail server port (by default, 110 for POP3).
Outgoing Mail Outgoing mail server name or IP, such as "smtp.server.com".
Outgoing Mail Port Outgoing mail server port (25 by default).
Requires SMTP Authentication If checked, SMTP authentication will be enabled by default.
Attachment Size Limit Specifies the maximum allowed attachment size in bytes.
Default User Charset Specifies the charset to be used by default.
Default User Time Offset Specifies the value of default time offset (in hours) relative to UTC. If this value is not specified, WebMail PHP uses the time offset, specified at the web server.
Path to WebMail PHP Attachment Folder Specifies a folder where users attachments will be temporarily stored.
New Administration Password Specifies WebMail Administrator password. Sets to default value "12345" after WebMail installation. It is strongly recommended to change it after installation. If these fields are left blank, existing password remains unchanged.
Confirm Password

 

Interface Settings section.

Value Description
Mails Per Page Specifies the maximum allowed number of mail messages to be displayed on web page.
Default Skin Specifies default skin for WebMail appearance.
Show Text Labels If checked, the corresponding text labels are displayed at all menu items.
Allow AJAX Version If this option is turned on and your browser supports required components, MailBee WebMail Lite interface will be switched to AJAX version.

 

Login Settings section.

Value Description
Standard Login Panel All fields are displayed (email, login, password).
Hide Login Field Hide login field and take it from email:
  1. Use Email as Login - whole email address is used as login (e.g. if email is jdoe@domain.com, login will be jdoe@domain.com)
  2. Use Account Name as Login - truncate email address from "@" and use the remainder (e.g. if email is jdoe@domain.com, login will be jdoe)
Hide Email Field Hide Email field and take it from logindomain to use - what domain will be displayed (option 1) and/or appended to login (option 2):
  1. Display Domain After Login Field - on login form, "@"domain will be displayed after login field. This parameter affects only visual look of login panel. Decision on how to build login or email address is not altered.
  2. If ticked, WebMail will construct email account login as concatenation of Login field contents + "@" + domain. You should use this if logins on your mail server look like jdoe@domain.com (i.e. they are the same as email address). Do NOT use this option if logins look like jdoe (e.g. just account name, without domain).
Allow Advanced Login If checked, clients of the WebMail system will be able to use the system with POP3/SMTP servers different from specified in POP3/SMTP Server settings.
Automatically detect and correct if user inputs e-mail instead of account-name. Allow WebMail to detect and fix when user inputs email instead of login, or login instead of email, and it's possible to determine correct login or e-mail from typed. Example: if user inputs full email address in Login field (while only account name is expected), WebMail will automatically drop @domain part.

 

Debug Settings section.

Value Description
Enable Logging Enable logging of mail operations (POP3/SMTP) into a file.
Path For Log Specifies physical location (e.g. C:\Data\log.txt) where to create the log.
View Log View log file.
Clear Log Clear log file.
Disable Error Handling Turn off automatic error handling in WebMail.
